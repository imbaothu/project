◆ audio_adsr_envelope()

None AudioProcessor.audio_adsr_envelope ( Array audio_data)

One technique to make the wave samples sound more natural is to apply an envelope to modify the amplitudes of the wave at different times. This function applies the attack/decay/substain/release (ADSR) envelope to the input audio_data. Ref: https://en.wikipedia.org/wiki/Envelope_(music). The number of attack, decay, and release samples is defined by adsr_attack_samples, adsr_decay_samples, and adsr_release_samples. The first adsr_attack_samples samples are in the attack phase, the next adsr_decay_samples samples are in the decay phase, and the last adsr_release_samples samples are in the release phase. Otherwise, the samples are in the substain phase. In each phase, the gain is computed differently. For each sample in audio_data (denote it as the i-th sample), this function first computes the current gain \(g(i)\) by \( g(i) = \begin{cases} \frac{1.2i}{\textbf{adsr_attack_samples}} & \text{if } i < \textbf{adsr_attack_samples} \\ 1 + \frac{0.2(\textbf{adsr_attack_samples} + \textbf{adsr_decay_samples} + i)}{adsr_decay_samples} & \text{if } \textbf{adsr_attack_samples} \leq i < \textbf{adsr_attack_samples} + \textbf{adsr_decay_samples} \\ 1 & \text{if } \textbf{adsr_attack_samples} + \textbf{adsr_decay_samples} \leq i < \text{length of audio data} - \textbf{adsr_release_samples} \\ \frac{\text{length of audio data} - 1 - i}{\textbf{adsr_release_samples}} & \text{if } \text{length of audio data} - \textbf{adsr_release_samples} \leq i < \text{length of audio data} \end{cases} \)

and then amplifies the current sample (audio_data[i]) by \(g(i)\). Note that if the lengh of audio_data is less than adsr_attack_samples + adsr_decay_samples + adsr_release_samples, this function calls audio_rise_fall_envelope() to apply the rise/fall envelope instead.

Parameters
audio_dataThe audio data.

◆ audio_generate_complex_wave()

None AudioProcessor.audio_generate_complex_wave ( Array audio_data,
float freq,
float amp,
int samples_per_sec )

Given an input frequency freq and an amplitude amp, this function samples the continuous complex sine wave function \(\mathrm{complex}\) to the input Array audio_data. The sampling rate is determined by the input samples per second samples_per_sec. For each sample in audio_data (denote it as the i-th sample), this function first computes the current time t (dividing i by samples_per_sec). Then, it computes the current angle theta (multiplying t by freq and \(2\pi\)). At last, it sets the i-th sample audio_data[i] to \(\mathrm{complex}\)(theta) * amp. In this project, we define the \(\mathrm{complex}\)(theta) using \(\mathrm{\sin}\)(theta) as follows: \( \mathrm{complex}(\theta) = \frac{\mathrm{sinwave}(\theta)}{\max(|\mathrm{sinwave}|)} \) where \(\mathrm{sinwave}\)(theta) is defined as: \( \mathrm{sinwave}(\theta) = \left( (\sin(\theta) + \frac{1}{2}\sin(2\theta) + \frac{1}{4}\sin(3\theta) + \frac{1}{8}\sin(4\theta) + \frac{1}{16}\sin(5\theta) + \frac{1}{32}\sin(6\theta)) \times \exp(-0.0004\theta)\right)^3 \) and \(\max(|\mathrm{sinwave}|)\) is the maximum of the absolute value of all \(\mathrm{sinwave}(\theta)\).

The sampling rate is determined by the input samples per second samples_per_sec. For each sample in audio_data (denote it as the i-th sample), this function first computes the current time t (dividing i by samples_per_sec). At last, it sets the i-th sample audio_data[i] to \(\mathrm{sawtooth}\)(sample_pos) * amp. Note that, one can compute \(\mathrm{sawtooth}\)(sample_pos) as follows: \( \mathrm{sawtooth}(p) = -1 + 2p \)

◆ audio_generate_string_wave()

None AudioProcessor.audio_generate_string_wave ( Array audio_data,
float freq,
float amp,
int samples_per_sec )

Given an input frequency freq and an amplitude amp, this function samples the continuous string function \(\mathrm{string}\), using the Karplus-Strong algorithm (https://en.wikipedia.org/wiki/Karplus%E2%80%93Strong_string_synthesis), to the input Array audio_data. The sampling rate is determined by the input samples per second samples_per_sec. In a word, the Karplus-Strong algorithm continuously and repeatedly averages consecutive random wave samples into the output string samples. In this project, this function implements a modified version of the Karplus-Strong algorithm as follows:

  • First, instead of using random samples, the function initializes an Array wave_samples of size samples_per_sec / freq, and calls audio_generate_square_wave() to create wave_samples.
  • Then, initialize prev_idx to the last index of the wave samples and cur_idx to 0.
  • For each sample in audio_data (denote it as the i-th sample),
    • set the current audio sample (audio_data[i]) to the average of the wave samples (wave_samples) at prev_idx and cur_idx.
    • set the wave samples at cur_idx to audio_data[i].
    • set prev_idx to cur_idx.
    • advance cur_idx by 1. If it is out of bounds, reset it to 0. Hint: You may handle the out-of-bound situation by using the modulo operator.

audio_dataThe audio data.
freqThe string wave frequency.
ampThe string wave amplitude.
samples_per_secThe number of samples per second.

float AudioProcessor.audio_note_number_to_freq ( int note_number)

Musical note numbers can be converted to their corresponding wave frequencies (Ref: https://www.inspiredacoustics.com/en/MIDI_note_numbers_and_center_frequencies). In short, given a note number note_number (denote by \(n\)), this function returns the corresponding wave frequency using this formula: \(440 \times 2^{\frac{n - 69}{12}}\).

◆ audio_rise_fall_envelope()

None AudioProcessor.audio_rise_fall_envelope ( Array audio_data)

One technique to make the wave samples sound more natural is to apply an envelope to modify the amplitudes of the wave at different times. This function applies the rise/fall envelope to the input audio_data. Let \(m\) be the index of the middle sample of audio_data. For each sample in audio_data (denote it as the i-th sample), this function first computes the current gain \(g(i)\) by \( g(i) = \begin{cases} \frac{i}{m} & \text{if } i <= m \\ \frac{\text{length of the audio data} - 1 - i}{\text{length of the audio data} - 1 - m} & \text{otherwise.} \end{cases} \) and then amplifies the current sample (audio_data[i]) by \(g(i)\).

Parameters
audio_dataThe audio data.

◆ audio_stereo_gains()

Tuple[float, float] AudioProcessor.audio_stereo_gains ( float angle)

This function computes appropriate gains for the stereo left and right channels, according to a pan angle angle specified in radians. It returns a tuple of two gains using the below formula. Given an angle \(\theta\), the left \(L\) and right \(L\) gains are: \(L = \frac{\sqrt{2}}{2} (\cos(\theta) + \sin(\theta))\) and \(R = \frac{\sqrt{2}}{2} (\cos(\theta) - \sin(\theta))\).

Parameters
angleThe pan angle between the two stereo sources.
Returns
A tuple of the left and right channel gains.

◆ audio_stereo_mix_in()

None AudioProcessor.audio_stereo_mix_in ( Array stereo_data,
Array channel_data,
int which_channel )

This function first checks if the number of stereo audio samples (stereo_data) is twice the number of input channel data (channel_data). If not, it raises an exception. Otherwise, for each sample in channel_data (denote it as the i-th sample), using the input which_channel, this function mixes channel_data[i] into stereo_data at the position i * 2 + which_channel, where which_channel equal to 0 is the left channel, and 1 is the right channel. When mixing it in, it adds a new value to the existing value.

Parameters
stereo_dataThe stereo audio data.
channel_dataThe mono channel audio data to mix in.
which_channelThe channel to mix in.
